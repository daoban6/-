## 🔧 步骤1：创建Supabase数据库

### 1.1 创建新项目
1. 打开 https://supabase.com/dashboard
2. 点击 **New Project** 按钮
3. 填写项目信息：
   - **Project name**: 输入项目名称（如 `辅料管理系统`）
   - **Database password**: 设置数据库密码（记住这个密码，后面可能用到）
   - **Region**: 选择离你最近的区域（如 `Asia Pacific (Singapore)`）
4. 点击 **Create new project**

### 1.2 获取API信息
1. 项目创建成功后，进入项目页面
2. 在左侧菜单点击 **Settings** > **API**
3. 复制以下信息（后面会用到）：
   - **Project URL**: 类似 `https://xxxx.supabase.co`
   - **anon public key**: 类似 `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`

## 🗄️ 步骤2：创建数据库表

### 2.1 打开SQL编辑器
1. 在Supabase左侧菜单点击 **SQL Editor**
2. 点击 **New query**

### 2.2 执行创建表的SQL
复制以下代码，粘贴到SQL编辑器中，然后点击 **Run**：

```sql
-- 创建用户表
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  name VARCHAR(100),
  role VARCHAR(20) DEFAULT 'employee',
  department VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建物料表
CREATE TABLE materials (
  id SERIAL PRIMARY KEY,
  code VARCHAR(50) UNIQUE NOT NULL,
  name VARCHAR(100) NOT NULL,
  spec VARCHAR(100),
  unit VARCHAR(20) NOT NULL,
  quantity DECIMAL(10,2) DEFAULT 0,
  warningValue DECIMAL(10,2) DEFAULT 0,
  status VARCHAR(20) DEFAULT 'normal',
  category VARCHAR(50),
  location VARCHAR(100),
  remark TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建领用记录表
CREATE TABLE usage_records (
  id SERIAL PRIMARY KEY,
  record_type VARCHAR(20) NOT NULL,
  contract_number VARCHAR(100),
  project_name VARCHAR(100),
  material_id INT REFERENCES materials(id),
  material_code VARCHAR(50),
  material_name VARCHAR(100),
  material_spec VARCHAR(100),
  material_unit VARCHAR(20),
  quantity DECIMAL(10,2) NOT NULL,
  user_name VARCHAR(100),
  date DATE NOT NULL,
  remark TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_materials_code ON materials(code);
CREATE INDEX idx_usage_records_date ON usage_records(date);

-- 插入初始超级管理员（密码：admin123）

```

> ⚠️ **注意**: 执行成功后，初始管理员账号：`admin`，密码：`admin123`

## 🚀 步骤3：部署到Vercel

### 3.1 创建GitHub仓库
1. 打开 https://github.com/new
2. **Repository name**: 输入仓库名（如 `auxiliary-material-system`）
3. 勾选 **Initialize this repository with a README**
4. 点击 **Create repository**

### 3.2 上传代码到GitHub
1. 打开项目文件夹 `辅料看板`
2. 复制所有文件到你的GitHub仓库
   - 可以用GitHub Desktop或直接在网页上传

### 3.3 在Vercel中导入项目
1. 打开 https://vercel.com/dashboard
2. 点击 **Add New** > **Project**
3. 选择你的GitHub仓库
4. 点击 **Import**

### 3.4 配置环境变量
1. 在Vercel项目配置页面，点击 **Settings** > **Environment Variables**
2. 添加以下变量：
   - **SUPABASE_URL**: 粘贴从Supabase复制的Project URL
   - **SUPABASE_KEY**: 粘贴从Supabase复制的anon public key
   - **JWT_SECRET**: 输入一个随机字符串（如 `my-secret-key-12345`）

3. 点击 **Deploy** 开始部署

### 3.5 获取访问地址
部署完成后，Vercel会给你一个访问地址，类似：
- `https://auxiliary-material-system.vercel.app/`

## 📖 步骤5：后续操作指南

### 5.1 添加员工账号
1. 登录系统后，点击右上角用户菜单
2. 选择 **用户管理**
3. 点击 **新增用户**
4. 填写用户信息：
   - 用户名：员工账号
   - 密码：初始密码
   - 姓名：员工姓名
   - 角色：选择权限级别
   - 部门：员工所属部门
5. 点击 **保存**

### 5.2 修改密码
1. 登录系统后，点击右上角用户菜单
2. 选择 **修改密码**
3. 输入原密码和新密码
4. 点击 **保存**

### 5.3 导入物料数据
1. 在主页面点击 **导入物料**
2. 选择Excel文件（格式：编码、名称、规格、单位、数量、预警值）
3. 点击 **导入**

### 5.4 导出数据
1. 在主页面点击 **导出物料** 导出所有物料
2. 在领用记录页面点击 **导出记录** 或 **导出统计报告**

### 5.5 查看数据
- **物料库存**: 在主页面直接查看
- **领用记录**: 点击顶部 **领用记录** 菜单
- **统计分析**: 在领用记录页面点击 **统计分析**

### 5.6 备份数据
1. 登录Supabase
2. 进入项目 > **Settings** > **Database**
3. 点击 **Backups**
4. 点击 **Create backup** 创建备份

## ❓ 常见问题

### Q1: 登录失败怎么办？
- 检查用户名密码是否正确（默认：admin/admin123）
- 检查网络是否正常
- 联系管理员重置密码

### Q2: 数据不同步？
- 刷新页面重试
- 检查网络连接
- 确保所有人都登录了同一个系统

### Q3: 忘记管理员密码？
1. 登录Supabase
2. 进入 **SQL Editor**
3. 执行以下SQL重置密码：
```sql
UPDATE users SET password = '$2a$10$N9qo8uLOickgx2ZMRZoMye.IjzqAKL9xL5jvMFVdNJHvGCgTq/VEq' WHERE username = 'admin';
```
密码将重置为 `admin123`

### Q4: 如何添加新的物料类别？
在添加物料时直接输入类别名称即可，系统会自动识别。

### Q5: 多人同时操作会冲突吗？
不会，系统采用云端数据库，实时同步数据，不会冲突。
