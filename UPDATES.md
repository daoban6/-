**数据库表结构**：
```sql
CREATE TABLE operation_logs (
  id SERIAL PRIMARY KEY,
  type VARCHAR(50) NOT NULL,
  file_name VARCHAR(255),
  details JSON,
  success BOOLEAN NOT NULL,
  message TEXT,
  operator_id INT REFERENCES users(id),
  operator_name VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_operation_logs_created_at ON operation_logs(created_at);
ALTER TABLE operation_logs DISABLE ROW LEVEL SECURITY;
```

### 2026-05-09

#### 16. 领料单日期显示错误
**问题描述**：领料单显示的创建时间与实际创建时间不符，如显示13:08实际应为21:08

**根本原因**：
- 数据库存储的是UTC时间（如 `2026-05-09T00:20:50.806808`），没有`Z`后缀
- JavaScript解析时将UTC时间误认为本地时间，导致时区转换错误

**解决方案**：
- 前端：通过添加`Z`后缀让JavaScript正确识别为UTC时间
- 使用 `created_at` 字段（数据库自动生成）代替手动设置的 `date` 字段
- 代码位置：`usage-tracking.html` 的 `formatDate` 函数

#### 17. 导出Excel日期格式问题

## 📊 操作日志说明

### 功能概述
系统提供操作日志功能，记录以下操作：
- Excel导入物料
- 快速更新库存

### 日志记录内容
| 字段 | 说明 |
|------|------|
| type | 操作类型（import/quick_update） |
| file_name | 导入的文件名 |
| details | 库存变化明细（JSON格式） |
| success | 是否成功 |
| message | 操作结果消息 |
| operator_id | 操作人ID |
| operator_name | 操作人姓名 |
| created_at | 创建时间 |

### 日志查看方式
1. 点击主页面的"操作日志"按钮
2. 在弹窗中查看所有操作记录
3. 支持分页显示
4. 支持清空日志（超级管理员权限）
